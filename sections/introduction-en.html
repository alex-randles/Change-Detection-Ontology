Introduction back to ToC

The Change Detection ontology represents changes that have occured in the source data which has created Linked data. Each change is represented as an event (lode:Event) which is specific to the change that has occured. A change is characterised based on the action which caused it. For example, if data has been deleted from the source data, the change will be represented as a deletion change (cdo:DeleteSourceData) Changes are grouped into change logs (cdo:ChangeLog) which relates to a specific source data and owner (cdo:hasOwner). Furthermore, information related to why (cdo:hasReason) the change has occured is captured.
The ontology allows for a notification policy to be defined using the Rei Policy Ontology. An example of 2 insert data changes and a notification policy is shown within this specification.
